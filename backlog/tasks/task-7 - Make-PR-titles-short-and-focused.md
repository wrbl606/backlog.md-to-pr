## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
The PR title generated by the workflow currently uses the first # heading from the task markdown as the {summary} token. This can produce long, unfocused titles like "Implement the user authentication flow with OAuth2 and session management across all endpoints".

The summary extraction should produce a concise, focused title that captures the core intent of the task.
